What's included

Accessibility guidelines

Comprehensive guidance for writing inclusive UX text that works for everyone.

  • Screen reader optimization
  • Cognitive accessibility
  • WCAG compliance
  • Multi-modal communication

Research-backed metrics

Quantifiable benchmarks based on comprehension research and industry standards.

  • Sentence length targets
  • Comprehension rates (8 words = 100%)
  • Reading level guidelines
  • Character limits by content type

Expanded error patterns

Four distinct error types with timing, location, and recovery guidance.

  • Validation errors (inline)
  • System errors (modal/banner)
  • Blocking errors (full-screen)
  • Permission errors

Tone adaptation framework

Map user emotional states to appropriate tone while maintaining consistent voice.

  • By emotional state
  • By content type
  • Context-specific examples
  • Voice chart templates

Real-world examples

Before/after improvements with scoring against quality standards.

  • E-commerce patterns
  • SaaS applications
  • Mobile experiences
  • Common anti-patterns

Fillable templates

Practical guides for creating consistent, high-quality UX text.

  • Error message templates
  • Empty state guides
  • Onboarding flows
  • Voice charts

Figma integration

Review and improve UX copy directly from your Figma designs. Perfect for content designers and product teams.

Quick setup

claude mcp add --transport http figma https://mcp.figma.com/mcp

Then share any Figma frame link with Claude to get instant UX writing feedback, accessibility audits, and specific improvements.

Get started in 3 steps

1

Download

Download the skill and extract the ZIP file to find the ux-writing folder.

2

Install

Copy the ux-writing folder to ~/.claude/skills/ and restart Claude Code.

3

Use

Ask Claude to review any interface copy and it will automatically apply UX writing best practices.
